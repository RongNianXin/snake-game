# 定制诗词与打油诗生成基本声韵规范 (Poetry Rhyme Rules)

本文件作为本贪吃蛇游戏 2.x/3.x 版本中，AI 生成自定义诗句、关卡文本、或吃错字引发的“打油诗惩罚机制”时的**核心规则底层库**。AI 必须严格遵守以下声韵、对仗及 ID 号段基本原则，以确保文本朗朗上口、数据结构规范。

---

## 🎵 一、 核心声韵规则（拒绝平庸读音）

汉语拼音的四个声调在传统诗词中分为**平（一声、二声）**和**仄（三声、四声）**。

### 1. 🎯 押韵与落脚点原则（绝对死律）
* **单句（上句）末尾字**：必须使用**仄声（三声 ˇ 或 四声 ˋ）**。
* **双句（下句）末尾字**：必须使用**平声（一声 ˉ 或 二声 ˊ）**，并且必须**押韵**。
* **【红线禁止】**：严禁出现上下句末尾字读音完全相同、或者同音不同字的情况（如：禁止“路”对“录”、“苍”对“舱”）。

> 👍 **正确示例**：
> 寄存器配错（cuò，仄声/上句尾）
> 通宵调测难（nán，平声/下句尾，两句字不同音）

### 2. 🌊 一、二句声调交替原则（平仄跌宕）
五言句式的核心是节奏感。为了避免读起来像流水账，上下句对应的位置声调应当尽量相反：
* **标准五言仄起平收格式**：
  * 上句：`仄 仄 平 平 仄`
  * 下句：`平 平 仄 仄 平`
* **AI 执行标准**：重点卡死**第 2 个字**和**第 5 个字（末尾字）**。如果上句第 2 字是平声，下句第 2 字必须是仄声。

---

## ✍️ 二、 对仗与词性规则（打油诗也需整齐）

无论是开发者自嘲、武侠还是经典题材，上下两句的结构必须对称：

| 位置 | 上句词性要求 | 下句词性要求 | 示例 (上句 vs 下句) |
| :--- | :--- | :--- | :--- |
| **第 1~2 字** | 名词 / 动词短语 | 相同结构的短语 | `电路` (名词) vs `芯片` (名词) |
| **第 3~4 字** | 形容词 / 动词短语 | 相同词性对仗 | `刚通电` (动补) vs `冒青烟` (动宾) |
| **第 5 字** | 核心动词 / 结尾名物 | 核心动词 / 结尾名物 | `电` (仄声) vs `烟` (平声) |

## 🤖 三、 面向 AI 的少样本提示与号段管理 (Few-Shot & ID Database Rules)

当需要 AI 扩展 `poetry-data.js` 词库或动态生成打油诗时，**必须严格遵守以下 ID 预留号段规范与题材定义**：

### 1. 🟢 墨香诗词阁 (category: "classic") —— 【ID号段：1 ~ 99】
* **题材内容**：经典传统五言绝句、古诗、对联。
* **合格范本**：`床前明月光，疑是地上霜。` (光 guāng / 霜 shuāng 押韵)

### 2. 🟡 刀光剑影录 (category: "martial") —— 【ID号段：101 ~ 199】
* **题材内容**：热血、豪放的武侠、修仙、江湖对仗诗句。
* **合格范本**：`刀出惊风雨，剑起震苍穹。` (雨 yǔ 仄尾 / 穹 qióng 平尾)

### 3. 🔴 开发者自嘲 (category: "develop") —— 【ID号段：201 ~ 299】
* **题材内容**：围绕软件代码、嵌入式硬件（单片机/寄存器）、电子电路（EDA画板/布线/炸板）、研发实验室、理工科科研通宵等场景的幽默自嘲。
* **硬件自嘲范本 1**：`电路刚通电，芯片冒青烟。` (电 diàn 仄尾 / 烟 yān 平尾)
* **软件自嘲范本 2**：`代码一行乱，通宵调测难。` (乱 luàn 仄尾 / 难 nán 平尾)

## 🛠️ 四、 玩家二创与 AI 自动跑测自检清单

如果你是玩家（或正在执行生成任务的 AI ），在保存数据文件前，请严格核对以下清单：
1. [ ] **ID检查**：我新加的这首诗，其 ID 是否在对应题材的预留号段内？（如：新增硬件梗，ID 必须是 204、205 等）
2. [ ] **上句尾音**：我写的上句最后一个字，读起来是不是往下沉的仄声（三声或四声）？
3. [ ] **下句尾音**：我写的下句最后一个字，读起来是不是平平的、可以拉长音的平声（一声或二声）？
4. [ ] **押韵检查**：上下句的最后一个字，绝对不是同音字，且拼音韵母相同（完美押韵）。
